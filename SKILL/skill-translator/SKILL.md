# Translator Skill

这个 skill 指导你如何进行专业的翻译工作，包括翻译策略、格式规范和质量保证。

## 何时使用此 Skill

- 需要翻译长文档或技术文档
- 需要保持特定格式（如 Markdown、代码）
- 需要术语一致性
- 需要多轮校对和优化

**注意**: 简单的单词或短句翻译无需使用此 skill，直接翻译即可。

## 翻译工作流程

### 第 1 步：分析源文本
1. 识别源语言和目标语言
2. 判断文本类型（技术文档、营销文案、日常对话等）
3. 识别专业术语和关键词
4. 检查是否包含代码、公式等特殊内容

### 第 2 步：制定翻译策略
根据文本类型选择策略：

**技术文档**:
- 术语使用标准翻译
- 保持代码块不翻译
- API 名称、函数名保持原文
- 添加必要的注释

**营销文案**:
- 注重语言的感染力
- 适应目标文化
- 保持品牌调性

**日常对话**:
- 口语化、自然
- 符合目标语言习惯

### 第 3 步：执行翻译
按段落或句子进行翻译，保持：
- 原文格式（段落、列表、标题等）
- 代码块完整性
- 链接和引用

### 第 4 步：格式化输出
使用以下标准格式输出翻译结果：

```
🌐 翻译结果
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
源语言: {语言}
目标语言: {语言}
文本类型: {类型}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【原文】
{原文内容}

【译文】
{翻译内容}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 术语对照（如有）:
- {术语1}: {翻译1}
- {术语2}: {翻译2}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**输出**:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
源语言: 中文
目标语言: 英文
文本类型: 技术术语

【原文】
机器学习模型训练

【译文】
Machine Learning Model Training
```

### 示例 2: 技术文档翻译（包含代码）

**用户**: 翻译这段文档：
```
使用 fetch 函数获取数据：
const data = await fetch(url);
这是一个异步操作。
```

**输出**:
```
🌐 翻译结果
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
源语言: 中文
目标语言: 英文
文本类型: 技术文档
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【原文】
使用 fetch 函数获取数据：
const data = await fetch(url);
这是一个异步操作。

【译文】
Use the fetch function to retrieve data:
const data = await fetch(url);
This is an asynchronous operation.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 翻译说明:
- 代码块保持不变
- "异步操作" 译为 "asynchronous operation"（标准术语）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 示例 3: 长文档分段翻译

**用户**: 翻译一篇长文档

**工作流程**:
1. 将文档分成多个段落
2. 逐段翻译，保持上下文连贯
3. 汇总所有翻译
4. 统一术语
5. 按标准格式输出

## 翻译质量检查清单

在输出翻译结果前，确保：
- ✅ 术语翻译一致
- ✅ 格式保持完整（Markdown、代码块等）
- ✅ 语句通顺自然
- ✅ 没有遗漏内容
- ✅ 专有名词正确处理（保留或翻译）

## 特殊内容处理规则

### 代码和命令
- 保持不变，不翻译
- 代码注释可以翻译

### 专有名词
- 产品名、公司名：保持原文
- 技术术语：使用标准翻译 + 英文原文（首次出现时）
- 人名：音译

### 链接和引用
- URL 保持不变
- 链接文本可以翻译

## 注意事项

1. **上下文理解**: 理解整体语境，避免逐字翻译
2. **文化适配**: 考虑目标语言的文化背景
3. **专业性**: 技术文档使用专业术语
4. **可读性**: 译文要符合目标语言的阅读习惯
